llm

A unified CLI and GUI wrapper that gives ollama, mlx-lm, and vllm-mlx a consistent interface. One command to run, stop, download, and inspect models across all three providers. Set a default once and llm run always works without arguments.

ollama mlx-lm vllm-mlx
ollama
Local model runtime with a built-in model registry. Interactive and API modes.
mlx-lm
Optimised inference via the MLX framework.
vllm-mlx
High-throughput inference with the vLLM MLX backend.
ℹ
Only one model may run at a time. Session state is persisted in ~/.llm/state.json so llm stop and llm ps always work without arguments.

Installation

From source

Requires Python 3.9 or newer. No other dependencies — llm.py uses only the standard library.

git clone <repo> cd llm python3 llm.py --help
ℹ
Pixi is also supported for a managed environment and is required for pixi run build. Run pixi install then use python3 llm.py as above.

Standalone app

Build a completely self-contained .app bundle and .dmg installer — no Python or Pixi required to run the result.

# produces dist/llm.app and dist/llm.dmg pixi run build
1

Install from the DMG

Open dist/llm.dmg and drag llm.app to Applications.

2

Add to your shell

Append to ~/.zshrc or ~/.bashrc, then reload.

3

Confirm

Run llm --help in a new terminal window.

# Add to ~/.zshrc export PATH="/Applications/llm.app/Contents/MacOS:$PATH" # Reload and verify source ~/.zshrc llm --help

Provider dependencies

Install whichever providers you need separately. llm provider info shows which are currently detected.

Provider Install Notes
ollama ollama.com Interactive and API server modes
mlx-lm pip install mlx-lm Inference API server
vllm-mlx pip install vllm Inference API server
HuggingFace CLI pip install huggingface-hub Required for llm download with mlx-lm / vllm-mlx

Commands

llm ls List all locally downloaded models from all providers.
llm ps Show the currently running model, port, base URL, and PID.
llm run [provider model] Start a model. Uses the configured default when called bare.
llm stop Stop the running model. No arguments needed.
llm default [provider model] Show the current default, or set a new one.
llm download <provider> <model> Download a model via ollama pull or huggingface-cli.
llm provider info Show executables, ports, base URLs, and install status per provider.
llm provider set <provider> <path> Set the executable path or pixi directory for a provider.

Run llm --help or llm <command> --help for full option details.

llm provider set

Configures the path to a provider executable or pixi environment directory. The setting is persisted in ~/.llm/config.json.

If the path is a directory containing pixi.toml, llm invokes the provider with pixi run from that directory. Otherwise it is treated as a direct executable.

# Set the ollama executable (default: /usr/local/bin/ollama) llm provider set ollama /opt/homebrew/bin/ollama # Set the mlx-lm executable (default: /opt/homebrew/bin/mlx_lm) llm provider set mlx-lm /opt/homebrew/bin/mlx_lm # Point vllm-mlx at a pixi environment directory llm provider set vllm-mlx /path/to/vllm-mlx

After setting a path, llm provider info reflects the new executable and llm run uses it immediately. For a pixi-managed provider the info output shows pixi run <provider> (cwd: <path>).

llm download

Downloads a model using the appropriate tool for each provider. Requires ollama or the HuggingFace CLI to be installed.

# ollama — uses: ollama pull llm download ollama llama3.2 # mlx-lm — uses: huggingface-cli download llm download mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # vllm-mlx — same HuggingFace download llm download vllm-mlx mlx-community/Mistral-7B-v0.1-4bit

llm default

Sets the provider and model that bare llm run will use. Configuration is saved to ~/.llm/config.json.

# Set a default llm default mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # Show the current default llm default Default provider: mlx-lm Default model: mlx-community/Llama-3.2-3B-Instruct-4bit

llm run

Starts a model server or interactive session. Only one model may run at a time — attempting to start a second will error with a reminder to llm stop first.

Any flag not recognised by llm is forwarded directly to the provider binary, so all provider-specific options work transparently.

Provider Behaviour Default port
ollama Starts an interactive session (ollama run) 11434
mlx-lm Starts an inference API server 8080
vllm-mlx Starts an inference API server 8080
# Run the configured default llm run # Run a specific provider + model llm run ollama llama3.2 llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # Override host and port llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit --host 0.0.0.0 --port 8081 # Provider-specific flags are forwarded automatically llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --port 8080 --chat-template chatml --max-tokens 2048 llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit \ --port 8080 --max-model-len 4096 --dtype float16

llm stop

Reads the state file, sends SIGTERM to the saved PID, and clears the state. For ollama, also calls ollama stop <model> to release GPU memory. No arguments are required.

llm stop Sent SIGTERM to PID 42107. Stopped mlx-lm: mlx-community/Llama-3.2-3B-Instruct-4bit

Examples

First-time setup

# 1. Check what's installed and what you already have llm provider info llm ls # 2. Download a model llm download ollama llama3.2 llm download mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # 3. Set a default so bare 'llm run' always works llm default mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit llm default # confirm # 4. Run it llm run # 5. Check what's running (in another terminal) llm ps # 6. Stop it llm stop

mlx-lm with custom flags

llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --host 0.0.0.0 \ --port 8080 \ --chat-template chatml \ --max-tokens 2048

vllm-mlx with custom flags

llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit \ --host 0.0.0.0 \ --port 8080 \ --max-model-len 4096 \ --dtype float16

ollama on a custom port

llm run ollama llama3.2 --host 0.0.0.0 --port 11434

mlx-lm :8080 → http://127.0.0.1:8080/v1
vllm-mlx :8080 → http://127.0.0.1:8080/v1

Model storage

ollama ~/.ollama/models
mlx-lm ~/.cache/huggingface/hub
vllm-mlx ~/.cache/huggingface/hub

Runtime files

~/.llm/state.json Active session — provider, model, PID, port

Build & distribute

pixi run build invokes PyInstaller with llm.spec to produce a self-contained GUI .app bundle from llm_gui.py, then wraps it in a distributable .dmg using macOS's built-in hdiutil.

⚠
The build task only runs on osx-arm64. Running it on Linux, Windows, or Intel Macs will exit with an error.

What it produces

Artifact Description
dist/llm.app Self-contained macOS app bundle. No Python or Pixi required to run.
dist/llm.dmg Distributable disk image. Drag llm.app to Applications.

How the binary works

PyInstaller bundles a Python interpreter alongside the llm_gui.py wrapper and wxPython. When running as a frozen binary, llm detects the sys.frozen flag and resolves python3 from PATH for mlx-lm invocations — ensuring mlx_lm is loaded from the user's own environment, not the bundled one.

pixi run build --- PyInstaller build --- ... --- Creating DMG --- ... Build complete. App bundle dist/llm.app DMG dist/llm.dmg To use from Terminal without installing: dist/llm.app/Contents/MacOS/llm --help To install system-wide (add to ~/.zshrc or ~/.bashrc): export PATH="dist/llm.app/Contents/MacOS:$PATH"
