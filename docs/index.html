llm

A unified CLI and GUI wrapper that gives ollama, mlx-lm, and vllm-mlx a consistent interface. One command to run, stop, download, and inspect models across all three providers. Set a default once and llm run always works without arguments.

ollama mlx-lm vllm-mlx
ollama
Local model runtime with a built-in model registry. Interactive and API modes.
mlx-lm
Optimised inference via the MLX framework.
vllm-mlx
High-throughput inference with the vLLM MLX backend.
ℹ
Only one model may run at a time. Session state is persisted in ~/.llm/state.json so llm stop and llm ps always work without arguments.

Installation

From source

Requires Python 3.9 or newer. No other dependencies — llm.py uses only the standard library.

git clone <repo> cd llm python3 llm.py --help
ℹ
Pixi is also supported for a managed environment and is required for pixi run build. Run pixi install then use python3 llm.py as above.

Standalone Dashboard app

Build a self-contained dashboard .app and .dmg — no Python or Pixi needed to run the result.

# produces dist/llm.app and dist/llm.dmg pixi run build
1

Install from the DMG

Open dist/llm.dmg and drag llm.app to Applications.

2

Launch

Double-click llm.app in Applications to open the dashboard.

Provider dependencies

Install whichever providers you need separately. llm provider info shows which are currently detected.

Provider Install Notes
ollama ollama.com Interactive and API server modes
mlx-lm pip install mlx-lm Inference API server
vllm-mlx pip install vllm Inference API server

Commands

llm ls List all locally downloaded models from all providers.
llm ps Show the currently running model, port, base URL, and PID.
llm run [provider model] [--ctx N] [--temperature F] Start a model. Uses the configured default when called bare. Accepts sampling parameters.
llm stop Stop the running model. No arguments needed.
llm default [provider model] Show the current default, or set a new one.
llm download <provider> <model> Download a model via ollama pull or huggingface-cli.
llm rm <provider> <model> Permanently delete a local model.
llm provider info Show executables, ports, base URLs, and install status per provider.
llm provider set <provider> <path> Set the executable path or pixi directory for a provider.

Run llm --help or llm <command> --help for full option details.

Usage

llm provider set

Configures the path to a provider executable or pixi environment directory. The setting is persisted in ~/.llm/config.json.

If the path is a directory containing pixi.toml, llm invokes the provider with pixi run from that directory. Otherwise it is treated as a direct executable.

# Set the ollama executable (default: /usr/local/bin/ollama) llm provider set ollama /opt/homebrew/bin/ollama # Set the mlx-lm executable (default: /opt/homebrew/bin/mlx_lm) llm provider set mlx-lm /opt/homebrew/bin/mlx_lm # Point vllm-mlx at a pixi environment directory llm provider set vllm-mlx /path/to/vllm-mlx

After setting a path, llm provider info reflects the new executable and llm run uses it immediately. For a pixi-managed provider the info output shows pixi run <provider> (cwd: <path>).

llm download

Downloads a model using the appropriate tool for each provider. Requires ollama or the HuggingFace CLI to be installed.

# ollama — uses: ollama pull llm download ollama llama3.2 # mlx-lm — uses: huggingface-cli download llm download mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # vllm-mlx — same HuggingFace download llm download vllm-mlx mlx-community/Mistral-7B-v0.1-4bit

llm default

Sets the provider and model that bare llm run will use. Configuration is saved to ~/.llm/config.json.

# Set a default llm default mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # Show the current default llm default Default provider: mlx-lm Default model: mlx-community/Llama-3.2-3B-Instruct-4bit

llm run

Starts a model server or interactive session. Only one model may run at a time — attempting to start a second will error with a reminder to llm stop first.

Any flag not recognised by llm is forwarded directly to the provider binary, so all provider-specific options work transparently.

Provider Behaviour Default port
ollama Starts an interactive session (ollama run) 11434
mlx-lm Starts an inference API server 8080
vllm-mlx Starts an inference API server 8080

Flags

Flag Default Description
--host 127.0.0.1 Bind address for the server
--port provider default Port number (overrides the provider default)
--ctx 65000 Context window length. Maps to --num-ctx (ollama), --max-tokens (mlx-lm), --max-model-len (vllm-mlx).
--temperature provider default Sampling temperature
--top-p provider default Top-p (nucleus) sampling
--top-k provider default Top-k sampling
--min-p provider default Min-p sampling
--repeat-penalty provider default Repetition penalty
--presence-penalty provider default Presence penalty (vllm-mlx only)
ℹ
Model parameter flags map automatically to the correct provider-specific CLI option. For ollama, a temporary custom model is created via ollama create so that PARAMETER directives can be applied — the model name is deterministic (SHA-1 hash of the params), making repeated runs idempotent.
# Run the configured default llm run # Run a specific provider + model llm run ollama llama3.2 llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # Override host and port llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit --host 0.0.0.0 --port 8081 # Set context window and model parameters llm run ollama llama3.2 --ctx 32768 --temperature 0.7 --repeat-penalty 1.1 llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --ctx 32768 --temperature 0.8 --top-p 0.9 --top-k 40 # Provider-specific flags are forwarded automatically llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --port 8080 --chat-template chatml llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit \ --port 8080 --dtype float16 --presence-penalty 0.5

llm stop

Reads the state file and stops whichever model is running. No arguments are required.

  • If a PID is stored — sends SIGTERM to the entire process group, catching child processes such as vllm workers spawned by pixi.
  • If no PID is stored (model was auto-detected externally) — finds the listening process on the saved port via lsof and signals it.
  • For ollama — also calls ollama stop <model> to release GPU memory.
llm stop Sent SIGTERM to PID 42107. Stopped mlx-lm: mlx-community/Llama-3.2-3B-Instruct-4bit

llm rm

Permanently deletes a local model.

Provider Action
ollama Calls ollama rm <model>
mlx-lm / vllm-mlx Removes the model directory from the shared HuggingFace cache (~/.cache/huggingface/hub). Deleting via either provider name removes the files for both.
llm rm ollama llama3.2 llm rm mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # same cache directory — removes for both mlx-lm and vllm-mlx llm rm vllm-mlx mlx-community/Mistral-7B-v0.1-4bit

Examples

First-time setup

# 1. Check what's installed and what you already have llm provider info llm ls # 2. Download a model llm download ollama llama3.2 llm download mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit # 3. Set a default so bare 'llm run' always works llm default mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit llm default # confirm # 4. Run it llm run # 5. Check what's running (in another terminal) llm ps # 6. Stop it llm stop

mlx-lm with custom flags

llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --host 0.0.0.0 \ --port 8080 \ --chat-template chatml \ --max-tokens 2048

vllm-mlx with custom flags

llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit \ --host 0.0.0.0 \ --port 8080 \ --max-model-len 4096 \ --dtype float16

ollama on a custom port

llm run ollama llama3.2 --host 0.0.0.0 --port 11434

Context window length

Use --ctx to set the context window. The default is 65000. The flag maps to the correct provider-specific option automatically.

# 65 000 token context (default) llm run ollama llama3.2 # Reduce to 32k to save memory llm run ollama llama3.2 --ctx 32768 llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit --ctx 32768 llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit --ctx 32768

Model parameters

Sampling parameters are mapped to the correct provider flag automatically. For ollama a temporary custom model is created via ollama create; for mlx-lm and vllm-mlx the flags are passed directly.

# ollama — lower temperature, mild repetition penalty llm run ollama llama3.2 --temperature 0.7 --repeat-penalty 1.1 # mlx-lm — nucleus sampling with top-k llm run mlx-lm mlx-community/Llama-3.2-3B-Instruct-4bit \ --temperature 0.8 --top-p 0.9 --top-k 40 # vllm-mlx — presence penalty (vllm-mlx only) llm run vllm-mlx mlx-community/Mistral-7B-v0.1-4bit \ --temperature 0.6 --presence-penalty 0.5

Dashboard

The dashboard is a native macOS GUI built with wxPython. Launch it from the CLI or by double-clicking llm.app.

llm gui

Models tab

Lists every locally available model across all providers — ollama models and HuggingFace models (shown once for each provider that can load them).

The top toolbar contains four buttons: Run selected, Download, Delete, and Refresh.

  • Single-click + Run selected — starts the model immediately using saved settings (host, port, context window, model parameters), or provider defaults if none are saved.
  • Right-click a model row for a context menu: Run model, Model options, and Delete model.
  • Double-click (or Model options) — opens the settings dialog. Configure host, port, context window, and model parameters: temperature, top-p, top-k, min-p, repeat penalty, presence penalty. Settings are saved to ~/.llm/config.json and reused on every future run.
  • Delete — permanently removes the selected model (llm rm under the hood). HuggingFace models are removed from the shared cache, deleting them for both mlx-lm and vllm-mlx.

Download dialog

Click Download to search HuggingFace for models.

  • Search — type a keyword and press Enter or click Search.
  • Filter tags — enter comma-separated tags (e.g. text-generation, 4-bit) to narrow results; each tag is sent as a separate HuggingFace API filter parameter (AND logic).
  • Sortable columns — click any column header (Model, Downloads, Likes, Size) to sort ascending or descending.
  • Size column — shows total repository size in human-readable form (B / KB / MB / GB).
  • Copy — copies the selected model ID to the clipboard.
  • Non-blocking download — a separate progress window shows real-time percentage and bytes transferred. The main dashboard remains fully interactive while downloading.

Status banner

Shows the currently running model with its provider, base URL, host:port, PID, and start time. When the model was started with custom parameters, a second detail line shows ctx = N temperature = N …. Updated every 2 seconds by probing the provider's live HTTP endpoint.

  • Stop — terminates the server (sends SIGTERM to the process group; falls back to port-based kill for externally-started models).
  • Refresh — immediately re-probes the live HTTP endpoints to verify current state, bypassing the 2-second polling interval.
ℹ
Closing the dashboard while a model is running shows a prompt: Stop & close, Keep running, or Cancel. Models that keep running can be stopped later with llm stop.

Providers tab

Displays the executable path, default port, base URL, and model directory for each provider — the same information as llm provider info.

Build & distribute

pixi run build invokes PyInstaller with llm.spec to produce a self-contained GUI .app bundle from llm_gui.py, then wraps it in a distributable .dmg using macOS's built-in hdiutil.

⚠
The build task only runs on osx-arm64. Running it on Linux, Windows, or Intel Macs will exit with an error.

What it produces

Artifact Description
dist/llm.app Self-contained macOS app bundle. No Python or Pixi required to run.
dist/llm.dmg Distributable disk image. Drag llm.app to Applications.

How the binary works

PyInstaller bundles a Python interpreter alongside the llm_gui.py wrapper and wxPython. When running as a frozen binary, llm detects the sys.frozen flag and resolves python3 from PATH for mlx-lm invocations — ensuring mlx_lm is loaded from the user's own environment, not the bundled one.
